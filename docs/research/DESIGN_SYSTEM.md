# Дизайн-система Complex

## 1. Розбір референсу (jabko.ua)

Важливо: домену `yabko.ua` не існує — реальний сайт Ябко це **jabko.ua**. Перевірив саме його.

**Що там реально працює (беремо підхід, не активи):**

| Прийом | Що робить | Беремо? |
|---|---|---|
| **Майже безкольоровий шасі** | Фон білий, текст чорний, хром інтерфейсу сірий. Колір з'являється ТІЛЬКИ там, де є гроші: ціна, знижка, кнопка. Око йде на товар, бо конкурувати за увагу нема з чим | **Так** — це прямо збігається з «сіро-чорним» від замовника |
| **Товар несе колір, а не UI** | Фото на чистому світлому полі, картка без рамки або з ледь помітною | **Так** |
| **Ціна — найбільший елемент картки** | Ціна візуально важча за назву товару. Стара ціна — сіра, закреслена, дрібна, ЗВЕРХУ від нової | **Так** — точно як на скріні `..._346` |
| **Бейдж знижки — абсолютна сума** (`-6 300 ₴`), не відсотки | Конкретна економія в гривнях читається сильніше, ніж «-8%» | **Так** — і на нашому референсі теж `-67 ₴` |
| **Темна кнопка «Купити»** | Висока контрастність, нейтральна | **Ні, змінюємо** — ТЗ вимагає, щоб акцент працював на кнопці «Купити». У нас кнопка = акцент |
| **Щільна сітка 4 колонки** | Багато товару в полі зору, менше скролу | **Так** |
| **Мультирівневе мегаменю з глибиною 3-4** | Каталог побутової техніки інакше не влазить | **Так** + міні-емблеми зліва (скрін `..._337`) |
| **Свотчі кольорів прямо на картці у видачі** | Видно варіанти, не заходячи в товар | **Так** |

**Чого у них немає, а нам треба:** фасетні фільтри промислового масштабу (у них каталог вужчий), калькулятор доставки, порівняння, статуси наявності. Це проєктуємо самі.

**Головний висновок:** нейтральна база + рівно ОДИН акцент — це не естетичний вибір, а функціональний. Якщо акцент є тільки на ціні, знижці, кнопці «Купити» й активному фільтрі, то акцент = «тут гроші / тут дія». Щойно ми пофарбуємо ним ще щось — сигнал зникає.

## 2. ПАЛІТРА

База — нейтральний сірий (zinc-подібний, злегка холодний). Акцент — **бренд-синій** (`#15558F` / `#4E9BE0`), похідний від логотипу.

> **Зміна від 20.07.2026.** Спочатку акцентом був димчастий помаранчевий `#C2410C` / `#FB7318` (Q10 у ROADMAP був відкритим питанням до замовника). Замовниця надіслала логотип — темно-синій вордмарк COMPLEX, виміряний колір `#0E3F68` — і попросила привести сайт до кольору лого. Питання Q10 закрите: **акцент синій**.

**Три речі, які варто знати чесно:**

1. У світлій темі кнопка має **білий текст на темно-синьому**, у темній — **майже-чорний текст на світло-синьому**. Це навмисно: один і той самий хекс не тримає 4.5:1 в обох темах. Брендовий `#15558F` на темному тлі `#0A0A0B` дає лише ~2.4:1 — нижче навіть UI-порогу 3:1, тому в темній темі акцент світлішає до `#4E9BE0`. `--brand-deep` при цьому лишається справжнім `#0E3F68` в обох темах: це поверхня топбару, а не текст. Інверсія напряму тексту — стандартна практика (Vercel, Radix), виглядає нормально.

Не використовуй `border-border` на формах.
3. Історично: заливку кнопки `#E85D04` **довелось відкинути** — білий текст на ньому дає лише 3.50:1; її замінили на `#C2410C`, а після переходу на бренд-синій — на `#15558F` (7.70:1, найбільший запас з усіх варіантів акценту, які розглядались).

### 2.3. Правило акценту (дисципліна)

Акцент `--primary` (`#15558F`) дозволений **рівно в 4 місцях**:
1. Кнопка «Купити» (заливка)
2. Ціна (тільки коли є знижка; звичайна ціна — `--price-regular`, чорна)
3. Бейдж знижки (`-67 ₴`)
4. Активний стан фасетного фільтра / чіп / фокус-ring

`--brand-deep` (`#0E3F68`) — **не п'яте місце для акценту**, а окрема роль: верхня смуга хедера й лого. Це фірмова поверхня, а не сигнал «тут дія». Не фарбувати ним кнопки — інакше топбар і кнопка «Купити» зіллються в одну пляму й акцент перестане читатись.

Все інше — сіре, чорне, біле. Знижка **ніколи не червона** (червоне = помилка). Бейдж «Уцінка» — нейтральна темна обводка, не колір.

**Виняток — маскот.** Лисичка асистента (`--fox` / `--fox-muted`) руда і під це правило не підпадає: це колір персонажа, а не акцент інтерфейсу. Всередині власної шкали дисципліна та сама — приглушений `--fox-muted` в idle/thinking, повний `--fox` лише в hover/talking. Бейдж непрочитаного повідомлення на віджеті — навпаки, `--primary`: він синій, бо це вже елемент інтерфейсу.

## 3. ТИПОГРАФІКА

### Шрифт: **Inter Variable**, self-hosted через `next/font/local`

Чому Inter: повна й **якісна кирилиця** (Inter проєктувався з кирилицею з першого дня — на відміну від багатьох гротесків, де кирилиця «приліплена»); має **tabular numerals** (`tnum`) — критично, бо ціни в колонках сітки не повинні «стрибати»; змінний шрифт = одна вага замість шести файлів.

Українська специфіка, яку Inter тримає добре: `і`, `ї`, `є`, `ґ` та апостроф (`'`) — саме на них ламаються дешеві шрифти. Резерв, якщо замовник схоче м'якше: **Manrope** (теж повна кирилиця).

**Без CDN.** Кладемо `.woff2` у репозиторій:

```ts
// frontend/app/fonts.ts
import localFont from 'next/font/local'

export const inter = localFont({
  src: [
    { path: './fonts/InterVariable.woff2',        weight: '100 900', style: 'normal' },
    { path: './fonts/InterVariable-Italic.woff2', weight: '100 900', style: 'italic' },
  ],
  variable: '--font-sans',
  display: 'swap',
  preload: true,
  fallback: ['system-ui', 'Segoe UI', 'Roboto', 'sans-serif'],
})
```

```css
/* globals.css */
@layer base {
  html { font-family: var(--font-sans), system-ui, sans-serif; }

  /* ЦІНИ, коди товару, кількість, таблиці порівняння — моноширинні цифри */
  .tnum, .price, table td, .product-code {
    font-variant-numeric: tabular-nums;
    font-feature-settings: 'tnum' 1, 'cv05' 1; /* cv05 = 'l' з хвостиком, менше плутанини */
  }
}
```

### 4.1. ProductCard

```
┌───────────────────────────────────┐
│ [-67 ₴]              ♡  ⚖        │  ← бейдж знижки (л.вгорі), обране+порівняння (пр.)
│                                   │     іконки: opacity 0 → 1 на hover, завжди видимі на touch
│         [ ФОТО 1:1 ]              │  ← object-contain, padding 12px, фон --card
│                                   │
│  ● ● ●                            │  ← свотчі кольорів (макс. 5 + «+3»), 16px
├───────────────────────────────────┤
│ ● В наявності                     │  ← статус: крапка 6px + text-xs
│ Мікрохвильова піч LG MS2042DB     │  ← text-sm/500, line-clamp-2, фікс. висота 40px
│                                   │
│ 785 ₴  ← закреслена, price-old    │
│ 718 ₴                             │  ← price-lg, колір --price
│ [🐾 6 платежів]                   │  ← бейдж «оплата частинами», outline
│                                   │
│ [    Купити    ]                  │  ← primary, повна ширина, h-10
└───────────────────────────────────┘
```

**Специфікація:**
- Контейнер: `bg-card`, `rounded-[--radius]`, `border border-border`, `p-3`. **Без тіні у спокої.**
- Hover: `border-transparent`, `shadow-lg`, `-translate-y-0.5`, `transition 150ms`. Картка піднімається — фото не зумиться (у побутовій техніці зум фото нічого не додає).
- Focus-visible: `ring-2 ring-ring ring-offset-2`.
- Уся картка — `<a>` на товар; кнопка «Купити» та іконки ♡/⚖ — вкладені кнопки зі `stopPropagation`.
- **Висота ціни зарезервована завжди** (`min-h`), навіть без старої ціни — інакше сітка «стрибає».
- Skeleton: ті самі розміри блоків, `bg-muted`, `animate-pulse`.

**Бейджі (лівий верхній кут, стек вертикально, max 2):**

| Бейдж | Стиль | Токени |
|---|---|---|
| Знижка `-67 ₴` | Заливка акцентом, білий текст, `rounded-md px-2 py-0.5 text-xs font-semibold` | `bg-primary text-primary-foreground` |
| **Уцінка** | Обводка, темний текст — щоб не конкурувати зі знижкою | `border border-foreground/30 bg-card text-foreground` |
| Новинка | Обводка нейтральна | `border-border text-muted-foreground` |

**Бейдж «оплата частинами»** (окремо, під ціною, не в кутку):
- `inline-flex items-center gap-1.5 rounded-md border border-border bg-card px-2 py-1`
- Лого банку 16px (SVG, чорно-білий) + кількість платежів `text-xs font-semibold`.
- На сторінці товару — з тултипом (умови, сума платежу). У сітці — без тултипа (не чіпляти hover на мобільному).

**Стан «Немає в наявності»:** фото `opacity-60 grayscale`, кнопка «Купити» → `secondary`, текст «Повідомити про наявність». Ціна лишається кольоровою (людина має бачити, скільки коштує).

### 4.4. SpecsTable (Характеристики | Опис)

Жорстка вимога з INPUTS §2. Дві колонки на ПК, стек на мобільному, обидві згорнуті.

```
┌── Характеристики ──────────┐  ┌── Опис ──────────────────┐
│ Основні:                   │  │ Мікрохвильова піч LG…    │
│ ┌────────────────────────┐ │  │                          │
│ │ Виробник          LG   │ │  │ Бренд: LG                │
│ │ Об'єм             20 л │ │  │ Тип: Мікрохвильова піч   │
│ │ Тип управління Сенсорне│ │  │                          │
│ └────────────────────────┘ │  │ [rich HTML + фото]       │
│ ░░░ градієнт-фейд ░░░      │  │ ░░░ градієнт-фейд ░░░    │
│ Усі характеристики ▾       │  │ Весь опис ▾              │
└────────────────────────────┘  └──────────────────────────┘
```

- Обгортка: `grid lg:grid-cols-2 gap-6`. На `<lg` — `grid-cols-1`, Характеристики **першими**.
- Заголовок секції: `text-h2`, під ним `border-b border-border`.
- Група (`Основні:`, `Габаритні розміри:`) — `text-h3`, `mt-6 first:mt-0`.
- Рядок: `grid grid-cols-[1fr_auto] gap-4 py-2.5`, зебра `even:bg-muted`, назва `text-muted-foreground`, значення `text-foreground font-medium text-right`.
- **КРИТИЧНО (INPUTS §2):** одиниця виміру клеїться до **значення**, не до назви. Рендер: `Висота` → `284 мм`. Ніколи `Висота (мм)` → `284`. У БД одиниця в окремій колонці — конкатенуємо на рендері: `${value} ${unit ?? ''}`.trim()
- Згорнуто: `max-h-[320px] overflow-hidden` + `::after` градієнт `from-transparent to-card` висотою 64px. Розгорнуто — `max-h-none`, фейд `hidden`.
- Тригер: `<button>` `text-sm text-primary`, `aria-expanded`, `aria-controls`. Не `<a href="#">`.
- **Опис — це rich HTML з фото** (INPUTS §2). Стилізуємо через `prose` (typography plugin), картинки всередині — `rounded-lg my-4 max-w-full h-auto`. Санітизація на бекенді.
- SEO: обидва блоки в DOM повністю (згортання — CSS/JS), щоб краулер бачив текст.

## 5. КОЛІРНЕ КОДУВАННЯ НАЯВНОСТІ

Ніколи не покладаємось на колір самотою — **колір + крапка + текст** (WCAG 1.4.1 «Використання кольору»).

| Статус | Крапка / текст | Підкладка (бейдж) | Контраст | Іконка |
|---|---|---|---|---|
| **В наявності** | `--stock-in` `#15803D` / `#4ADE80` | `--stock-in-bg` `#ECFDF3` / `#0D2318` | 5.02:1 / 10.56:1 | заповнене коло `●` |
| **Під замовлення** | `--stock-order` `#B45309` / `#FBBF24` | `--stock-order-bg` `#FFF7ED` / `#2A1D06` | 5.02:1 / 11.02:1 | коло-контур `◐` (годинник) |
| **Немає** | `--stock-out` `#5C5C66` / `#A1A1AA` | `--stock-out-bg` `#F4F4F5` / `#26262A` | 6.61:1 / 7.18:1 | порожнє коло `○` |

**Форма подачі:**
- **У картці сітки:** крапка 6px + `text-xs` без підкладки — щоб не змагатись із бейджем знижки.
- **На сторінці товару:** повний бейдж із підкладкою, `rounded-full px-2.5 py-1 text-xs font-medium`, + уточнення поруч (`Під замовлення · 3–5 днів`).
- Зелений навмисно **приглушений** (`#15803D`, не `#22C55E`) — щоб не сперечався з акцентом. Наявність — фонова інформація, дія — це «Купити».
- «Під замовлення» — бурштиновий, а не червоний: товар **можна купити**, це не помилка.

## 6. Що ще зафіксовано

- **Радіуси:** `--radius: 10px`. Кнопки/інпути `8px`, картки `10px`, бейджі/чіпи `9999px` (pill), модалки `12px`.
- **Тіні:** тільки на підняття (hover картки, дропдауни, Sheet). Плоский інтерфейс у спокої.
- **Spacing:** база 4px. Сітка контенту `max-w-[1440px] px-4 lg:px-8`.
- **Фокус:** `ring-2 ring-ring ring-offset-2` — глобально, ніколи не `outline: none` без заміни.
- **Motion:** 150 мс (мікро), 200 мс (панелі), `ease-out`. Поважати `prefers-reduced-motion`.
- **Тема:** `next-themes`, `class`-стратегія, дефолт — **світла**, перемикач у хедері й у бургері. Темна тема — не декор: `<html class="dark">` ставиться до гідратації (inline-скрипт), щоб не було спалаху білим.
